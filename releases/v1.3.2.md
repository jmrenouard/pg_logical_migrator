# Release v1.3.2

## [1.3.2] - 2026-04-17

### Bug Fixes
- **migrator**: fix potential SQL injection in `setup_reverse_replication` by using parameterized queries for subscription existence checks.
- **migrator**: refactor `cleanup_reverse_replication` for robustness: implement best-effort `DROP` operations, granular error tracking, and detailed logging with exception info.
- **tui**: fix layout issues and improve color consistency across different terminal themes.
- **cli**: fix `KeyError` in progress monitoring and correct missing arguments during pipeline unpacking.
- **cli**: update `pg_migrator_completion.sh` with latest commands (`setup-reverse`, `cleanup-reverse`, `sync-lobs`, `init-replication`, `post-migration`).

### Features
- **migrator**: implement **Large Objects (LOBs/BLOBs) synchronization** (`sync-lobs`). This feature manually exports LOBs from the source and imports them into the destination, updating OID references via Primary Key matching to overcome logical replication's native OID limitation.
- **tui**: add "Sync Large Objects" step to the interactive dashboard and integrate it into the automated post-migration pipeline.
- **docs**: comprehensive documentation overhaul across the entire project for professional engineering clarity and technical precision.
- **docs**: add stylized Mermaid architecture diagram to the root README for better system visualization.
- **docs**: create a centralized documentation hub in `DOCS/README.md` with categorized technical references.
- **demo**: record and integrate a high-quality English `asciinema` demonstration showing the complete 17-step migration lifecycle.
- **governance**: add community-standard `CONTRIBUTING.md` and `SECURITY.md` files to define project collaboration and disclosure policies.
- **github**: implement structured GitHub Issue Templates for bug reports and feature requests.
- **packaging**: comprehensive E2E packaging tests integrated directly into the `make test-all` test suite, covering AlmaLinux 8/9, Ubuntu 22.04, and Python wheel generation.

### Maintenance
- **config**: update default test environment configuration in `config_migrator.ini` to use stable Docker service names (`pg_source`, `pg_target`) instead of dynamic IP addresses.
- **tests**: achieve **100% test coverage** for CLI commands (`test_cli_commands.py`) and pipelines (`test_cli_pipelines.py`).
- **tests**: implement robust async UI test suite for Textual (`test_tui.py`) using `pytest-asyncio`, achieving 99% coverage.
- **tests**: add comprehensive failure-path unit tests for `cleanup_reverse_replication`.
- **tests**: add dedicated unit tests for Large Object synchronization.
- **tests**: update E2E migration test suite to verify binary data (LOB) integrity.
- **tests**: eliminate leftover temporary test output files (`tests.out`) for cleaner local environment runs.
- **readme**: modernize project badges and add sponsorship links (Buy Me A Coffee).


### Workflow Synchronization
- **Standardization**: Fully synchronized the CLI, TUI, and documentation with the official **17-step migration lifecycle**.
- **CLI Alignment**: Re-numbered and renamed all CLI subcommands to match the four phases (Prepare, Replicate, Finalize, Audit).
- **Pipeline Expansion**: The `post-migration` automated pipeline now executes the full suite of finalization steps sequentially, including LOB synchronization, ownership reassignment, and comprehensive object/row audit checks.
- **TUI Synchronization**: Updated the interactive dashboard to reflect the new step numbering and enhanced the automated pipelines within the TUI to match the CLI behavior.
- **Documentation**: Verified and updated `DOCS/WORKFLOW.md` and `DOCS/TOOLS.md` to ensure absolute consistency with the implementation.
