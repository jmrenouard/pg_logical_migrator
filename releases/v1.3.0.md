# Release v1.3.0

## [1.3.0] - 2026-04-12

### Features
- **rollback**: add `setup-reverse` command to establish reverse replication (Destination -> Source) for fast rollback
- **monitoring**: improve `repl-status` to detect and display replication flows on both source and destination
- **workflow**: harmonize and align all user interfaces (CLI, TUI, Docs) on a standardized **17-step renumbered workflow**
- **documentation**: add **Mermaid diagrams** to `DOCS/WORKFLOW.md` for visual pipeline tracking
- **packaging**: add Python Wheel (`.whl`) and Source Distribution (`.tar.gz`) support via `pyproject.toml`
- **ci/cd**: add automated multi-platform binary generation (Linux, Windows `.exe`, macOS Apple Silicon)
- **packaging**: add native OS package generation for Debian (`.deb`) and RHEL/Fedora (`.rpm`) using `nfpm`
- **ci/cd**: automate GitHub pre-releases for any branch matching `v*` pattern
- **shell**: add bash completions script (`pg_migrator_completion.sh`) for easier CLI usage

### Bug Fixes
- **post-sync**: fix ownership reassignment to correctly handle and exclude automatic table-row types
- **rollback**: improve `setup-reverse` robustness with better pre-cleanup of replication slots
- **monitoring**: fix table name matching in replication progress tracking for `public` schema
- **tui**: fix rendering errors (`NotRenderableError`) by standardizing Rich Panel usage
- **tui**: fix Python < 3.12 syntax errors in f-string expressions (backslash handling)
- **tui**: fix missing `textwrap` import

### Refactor
- **cli**: modularize monolithic script into `src/cli/` sub-modules (`commands`, `pipelines`, `helpers`)
- **schema**: refine `pre-data` and `post-data` separation into distinct logical steps

### Docker
- **compatibility**: update `Makefile` to use `docker compose` for improved server environment support
- Updated images available on [Docker Hub](https://hub.docker.com/r/jmrenouard/pg_logical_migrator) and [GHCR](https://github.com/jmrenouard/pg_logical_migrator/pkgs/container/pg_logical_migrator)
