# Release v1.4.1

## [1.4.1] - 2026-05-06

### CI / CD & Packaging
- **packaging**: centralized `PYINSTALLER_OPTS` configuration in the `e2e_packaging_test.sh` script to explicitly adhere to the PyInstaller build requirements enforced in GitHub actions.
- **ci**: updated `.github/workflows/python-package.yml` to fully adopt the new containerized tests by triggering `make test-all` directly in the pipeline, unifying the local and remote continuous integration mechanisms.

### Tests
- **tests**: containerized the E2E and integration test suites using Docker Compose, establishing an isolated test environment (Runner + Database nodes) directly managed by the `Makefile`.
- **tests**: resolved execution permission blockages (`RESULTS/coverage` root ownership error) emerging from the new containerized test runners on local development machines.
- **tests**: recalibrated the `pytest-cov` threshold checks within the `Makefile` (`make test-coverage`) to gracefully handle the separation between unit test coverage and E2E coverage scopes.
